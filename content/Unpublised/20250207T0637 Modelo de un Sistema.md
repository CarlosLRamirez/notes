## Concepto Principal
El **modelo de un sistema** se define como una representación del sistema con el fin de estudiarlo. 

Un modelo es una **aproximación** al sistema real, es decir una representación **simplificada** y abstracta que captura las características y comportamientos clave que se desean estudiar.  Por otro lado, el **modelo** debe ser lo suficientemente detallado para permitir obtener conclusiones validas del sistema real que se esta estudiando.

Dicha representación puede estar dada mediante herramientas analíticas, algoritmos u otras técnicas. Los modelos utilizados para simular un sistema se llaman **modelos de simulación**.

## Contexto
Muchas veces no se puede experimentar con un sistema real, como un banco o una autopista., en ese caso lo mejor es experimentar con un **modelo** del sistema real mediante un ejercicio de  **simulación**.
## Aspectos Clave
- Así como un sistema posee diferentes componentes como  entidades, atributos y actividades, un modelo tambien debe representar dichos componentes, sin embargo el modelo únicamente contendrá aquellos que son relevantes para el estudio.
- El propósito de modelar un sistema es entender, analizar o predecir el comportamiento del sistema.
- Cuando construimos un modelo, siempre existe la pregunta sobre si refleja con precisión el sistema para los propósitos del estudio, es por eso aborda en las diferentes técnicas de validación de un modelo.

## Ejemplos
El **modelo** de un call center el cual podría incluir llegada de clientes, tiempo de servicio y longitud en cola de espera.

## Mis Conclusiones
Un modelo es un concepto muy abstracto, el reto es como representar un sistema real mediante algoritmos o código de programación
