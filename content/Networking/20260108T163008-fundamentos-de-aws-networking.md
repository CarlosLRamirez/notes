# AWS Networking Fundamentals - Índice Temático

**Duración:** 40 horas (13-14 semanas, 2 sesiones de 1.5h/semana)

---

## PARTE 1: FUNDAMENTOS DE AWS Y CONCEPTOS BASE (12 horas - 4 semanas)

### **Módulo 1: Introducción a Cloud Computing y AWS** (3 horas)

- Conceptos de cloud computing desde perspectiva de redes
- Infraestructura global de AWS: Regiones, AZs, Edge Locations
- Modelo de responsabilidad compartida (enfoque networking)
- Comparación: Datacenter tradicional vs. AWS
- Caso de uso: Arquitectura de red en premises vs. AWS

### **Módulo 2: Fundamentos de AWS para Profesionales de Redes** (3 horas)

- Consola de AWS y navegación
- AWS CLI básico para networking
- Conceptos de IAM aplicados a redes (políticas, roles)
- Documentación de AWS y recursos de aprendizaje
- Lab: Configuración inicial de cuenta y AWS CLI

### **Módulo 3: Modelo de Precios y TCO para Infraestructura de Red** (3 horas)

- Modelo de precios de servicios de red en AWS
- Costos de transferencia de datos
- Calculadora de precios AWS
- Optimización de costos en networking
- Caso práctico: TCO de migración de red

### **Módulo 4: Servicios de Cómputo Básico** (3 horas)

- Amazon EC2: Conceptos esenciales desde perspectiva de red
- Tipos de instancias y sus características de red
- Security Groups como firewall stateful
- Elastic IP y direccionamiento público
- Lab: Lanzar instancia EC2 y configurar conectividad básica

---

## PARTE 2: AWS NETWORKING PROFUNDO (28 horas - 9-10 semanas)

### **Módulo 5: Amazon VPC - Fundamentos** (4.5 horas)

- Arquitectura de VPC: Similitudes y diferencias con VLANs
- CIDR, subnetting en AWS
- Subredes públicas vs. privadas
- Internet Gateway y NAT Gateway
- Route Tables: Comparación con routing tradicional
- Lab: Crear primera VPC con subredes públicas/privadas

### **Módulo 6: Amazon VPC - Routing y Conectividad Avanzada** (4.5 horas)

- Route Tables avanzadas
- VPC Peering: Topologías y limitaciones
- Transit Gateway: Hub-and-spoke en AWS
- VPC Endpoints (Gateway y Interface)
- Lab: Implementar VPC peering y routing multi-VPC

### **Módulo 7: Seguridad de Red en AWS** (4.5 horas)

- Security Groups vs. Network ACLs
- Defensa en profundidad (layered security)
- VPC Flow Logs para análisis de tráfico
- AWS Network Firewall
- Bastion Hosts y Session Manager
- Lab: Implementar arquitectura de seguridad multi-capa

### **Módulo 8: Conectividad Híbrida** (4.5 horas)

- AWS VPN: Site-to-Site y Client VPN
- AWS Direct Connect: Conceptos y casos de uso
- Virtual Private Gateway vs. Transit Gateway para híbrido
- Routing BGP en AWS
- Lab: Configurar Site-to-Site VPN

### **Módulo 9: Balanceo de Carga y Alta Disponibilidad** (3 horas)

- Elastic Load Balancer: ALB, NLB, GLB
- Comparación con balanceadores tradicionales
- Health checks y distribución de tráfico
- Cross-Zone Load Balancing
- Lab: Implementar arquitectura con ELB

### **Módulo 10: DNS y Gestión de Tráfico** (3 horas)

- Amazon Route 53: DNS managed service
- Políticas de routing: simple, weighted, latency, geolocation, failover
- Route 53 Resolver para DNS híbrido
- Integración con otros servicios AWS
- Lab: Configurar Route 53 con políticas de routing

### **Módulo 11: Content Delivery y Edge Networking** (3 horas)

- Amazon CloudFront: CDN global
- Edge locations y caching
- AWS Global Accelerator
- Optimización de latencia y performance
- Caso práctico: Diseño de distribución global de contenido

### **Módulo 12: Monitoreo, Troubleshooting y Optimización** (3 horas)

- VPC Flow Logs análisis avanzado
- CloudWatch para métricas de red
- AWS Network Manager
- Reachability Analyzer
- Troubleshooting común de conectividad
- Lab: Diagnosticar y resolver problemas de red

### **Módulo 13: Arquitecturas de Referencia y Mejores Prácticas** (3 horas)

- Well-Architected Framework (pilar seguridad y confiabilidad)
- Patrones de arquitectura de red comunes
- Diseño de arquitecturas multi-tier
- Diseño de arquitecturas multi-región
- Caso práctico: Diseñar arquitectura completa

---

## PROYECTO FINAL INTEGRADOR (Semana final - no cuenta en las 40h de contenido)

**Diseño e Implementación de Arquitectura de Red Empresarial**

- Aplicar todos los conceptos aprendidos
- Documentación técnica de arquitectura
- Implementación práctica en AWS
- Presentación de proyecto

---

## Características del programa:

**Distribución horaria por sesión:**

- **Sesión 1 (1.5h):** Teoría + Demostración (60% teoría, 40% demo)
- **Sesión 2 (1.5h):** Laboratorio práctico + Troubleshooting (20% teoría, 80% práctica)

**Elementos adicionales:**

- **Evaluación continua:** Quizzes al final de cada módulo
- **Laboratorios:** 15-20 labs prácticos guiados
- **Recursos:** Acceso a AWS Free Tier, documentación oficial, arquitecturas de referencia
- **Material complementario:** Diagramas de red, comparativas CCNA-AWS, cheat sheets
