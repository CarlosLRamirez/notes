## Main Concept

AWS Trainium is a custom ML chip designed and built by AWS specifically for deep learning training on very large models. It is available as EC2 instances of type Trn1 and delivers significant cost savings compared to standard GPU instances for training workloads.

> [!TIP] Key Idea
> 
> - **Purpose** → training large deep learning models (100 billion+ parameters).
>     
> - **Instance type** → Trn1 (EC2 instances powered by Trainium chips).
>     
> - **Cost benefit** → up to 50% cost reduction compared to equivalent GPU-based training.
>     
> - **Environmental benefit** → lowest environmental footprint among ML instance types due to higher efficiency.
>     

## When to Use

```
Use Trainium when:
- Training very large deep learning models directly on EC2.
- Cost of training on standard GPU instances is prohibitive.
- You want the lowest environmental footprint for training workloads.
- You are not using SageMaker and need direct hardware control.
```

> [!example] Example
> 
> A research team is training a 200 billion parameter foundation model from scratch. Using standard GPU instances would cost $2M. Switching to Trn1 instances powered by Trainium reduces the cost to approximately $1M — same training task, 50% lower cost.

## Key Numbers for the Exam

```
Trn1 instance    → up to 16 Trainium accelerators per instance
Cost reduction   → up to 50% vs standard GPU instances for training
Model scale      → designed for 100 billion+ parameter models
Footprint        → lowest environmental footprint among ML instance types
```

## Critical Distinction: Trainium vs Inferentia

```
AWS Trainium    → for TRAINING large models
                  Trn1 instances
                  50% cost reduction on training

AWS Inferentia  → for INFERENCE (serving predictions)
                  Inf1 / Inf2 instances
                  70% cost reduction on inference
                  4x throughput vs GPU instances
```

> [!TIP] Key Exam Rule
> 
> - **Training a model** → Trainium.
>     
> - **Serving a model / making predictions** → Inferentia.
>
