# Github Repository

## Concepto Principal
El repositorio de Gihub es nuestro repositorio de git que hacemos push o subimos a Github, es una copia de nuestro repositorio local (working tree y repositorio).

## Contexto
Es la parte medular de Github, es la forma en que Github nos  permite acceder de forma visual y manejar nuestros los repositorios de git con varias funcionalidades adicionales.

Al crear una cuenta de Github podemos tener repositorios ilimitados "almacenados" en nuestra cuenta.

## Aspectos Clave
Entre las principales funcionalidades tenemos:
- Ver branches: para organizar diferentes versiones del código
- Ver tags: para marcar puntos importantes en la historia del proyecto
- Ver el historial de commits
- Explorar los archivos del repositorio: nos permite incluso editar desde la pagina web.
- Ver releases: : para manejar versiones o hitos de nuestro proyecto
- Ver la distribución del lenguaje utilizado en nuestro código
- Ver los archivos markdown que es encuentran en el primer nivel de una ruta:
	- Readme.md: Generalmente aqui es donde guardamos la documentación del repositorio
	- License.md: Este archivo contiene la información de los permisos legales para usar el proyecto.

Dentro del repositorio de Github podemos realizar estas acciones:
- Pinning repos: para marcar importante repositorios de nuestro perfil de Github
- Watching repos: Suscribirse a actualizaciones y cambios en repositorios de otros.
- Forking: Crear nuestra propia copia de algún proyecto de alguien y trabajar independiente en nuestra propia version. 
- Staring: Marcar algunos repositorios que nos interesen
- Cloning: Descargar una copia local del repositorio para trabajar en el.

Github nos permite tener repositorios tanto públicos como privados.
